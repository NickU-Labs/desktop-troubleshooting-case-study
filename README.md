# Desktop Troubleshooting Case Study

This repository documents real-world desktop PC hardware and Windows troubleshooting scenarios.

The goal is to demonstrate a **structured, repeatable IT support troubleshooting methodology**
used in help desk, desktop support, and junior sysadmin roles.

---

## Topics Covered

- RAM isolation and memory-related boot failures
- GPU detection issues and driver recovery
- Windows Device Manager and Task Manager diagnostics
- Display and monitor troubleshooting
- BIOS and hardware-level isolation techniques
- Windows Event Viewer analysis

---

## Troubleshooting Approach

1. Establish a known-good baseline
2. Change only one variable at a time
3. Observe symptoms carefully
4. Verify fixes using system tools
5. Document outcomes for repeatability

---

## Repository Structure

- `case-studies/` – Detailed troubleshooting walkthroughs
- `checklists/` – Reusable diagnostic checklists
- `lessons-learned.md` – Key takeaways from troubleshooting
